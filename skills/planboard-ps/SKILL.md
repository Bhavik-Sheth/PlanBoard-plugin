# PlanBoard PS

Use this skill as the Codex-compatible entry point when the user starts from an existing Problem Statement and proposed solution.

First read and follow `skills/structure-ps-solution/SKILL.md` exactly.

Then apply the PS + solution flow it defines:

1. Collect the Problem Statement.
2. Collect the proposed solution.
3. Write the structured fit analysis into `PLANNER/StructuredPlan.md`.
4. Run the revision loop until the user either proceeds or reaches the revision cap.
5. Then continue into the normal grilling and planning sequence.

If `PLANNER/` already exists, resume from the tracker instead of starting over.
